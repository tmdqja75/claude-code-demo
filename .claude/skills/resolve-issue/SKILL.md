# 깃허브 이슈 해결하기
너는 지금부터 깃허브 이슈를 해결하는 엘리트 소프트웨어 개발자야. 아규먼트로 깃허브 이슈 고유 숫자를 넘겨주면 이 숫자에 해당되는 Issue를 해결하는 작업을 해줘

## 작업 순서
- **이슈 불러오기**: 깃허브 CLI를 사용해서 이슈를 불러와줘. 제목, 본문, 레이블을 확인하고, 이슈를 정확히 이해해줘.
- **브랜치 세팅하기**: 이슈를 해결할 브랜치를 세팅해줘. dev 브랜치에서 issue-$ISSUE_NUMBER 형태로 새로운 브랜치를 생성 후 작업 진행해줘 (ex: issue-12)
- **코드베이스 분석하기**: 배정된 이슈를 해결하기 위해 알고 있어야 하는 정보를 분석하기 위해 코드 베이스를 분석하고 컨텍스트를 확보해줘. 최대 10개의 서브에이전트를 필요한 만큼 생성해서 독립적으로 코드베이스를 리서치 해줘.
- **이슈 해결 계획 세우기**: 취합한 정보를 기반으로 어떻게 이슈를 해결할지 Plan 모드를 사용해서 계획을 세워줘. 계획을 세우고 TODO리스트 수립이 완료되면 나한테 먼저 확인을 받아야 돼. 계획을 바로 실행하지 말고, 나와 상의하면서 수정 후 실행해야 해. 모호한 부분이 있으면 나한테 질문을 해줘
- **이슈 해결하기**: 계획 수립과 수정이 완료되었으면, 차례대로 계획을 실행해줘. 그리고 각 체크리스트 (todo list)가 완료 될 때마다 변경사항들을 요약한 커밋 메세지를 작성하고, 커밋을 만들어줘. 커밋 메세지는 한국어로 작성해줘
- **PR 생성하기**: 작업관련 된 내용이 포함된 풀 리퀘스트를 작성해줘. PR은 항상 새로운 브랜치를 메인 브랜치로 merge 해야돼. PR은 한국어로 작성해줘
